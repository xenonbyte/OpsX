<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI stdout/stderr -> machine consumer | any stray human text on stdout or unexpected stderr usage breaks JSON parsing and automation |
| persisted workspace state -> JSON envelope | expected blocked or missing states can be misclassified as transport failures if the CLI throws instead of serializing diagnostics |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-03 | Information Disclosure | `lib/cli.js` JSON output | mitigate | Force JSON mode to write only the serialized envelope to stdout and keep stderr reserved for exceptional failures. |
| T-08-04 | Tampering | `lib/cli.js` status branch | mitigate | Serialize expected missing/blocked states inside `workspace`, `migration`, `changeStatus`, `warnings`, and `errors` with exit `0` instead of throwing. |
</threat_model>
