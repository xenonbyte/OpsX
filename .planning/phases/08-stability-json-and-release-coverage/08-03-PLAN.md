<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| filesystem-derived artifact paths -> shared matcher helpers | literal `*`, `[`, `]`, or `?` characters can be misread as patterns if the repo does not escape them deterministically |
| normalized relative paths -> scope and hash logic | different normalization rules across modules can cause inconsistent allow/forbid decisions and drift reports |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-05 | Tampering | `lib/path-utils.js` / read-only consumers | mitigate | Centralize normalization and base-relative calculations once and make read-only consumers import the shared helpers instead of duplicating them. |
| T-08-06 | Elevation of Privilege | `lib/glob-utils.js` / `lib/path-scope.js` | mitigate | Escape literal glob characters and keep one `picomatch@4.0.4` wrapper so special characters cannot widen matching unexpectedly. |
</threat_model>
