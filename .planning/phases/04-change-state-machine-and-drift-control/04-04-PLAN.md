<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| artifact content -> stored hashes | silent hash refresh can accept unreviewed scope changes |
| state.yaml -> context/drift sidecars | mismatched sidecars can mislead resume/apply guidance |
| apply guidance -> task execution scope | more than one top-level task group can be consumed in one run |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-02 | Tampering | hash comparison and accepted writes | mitigate | Split `detectArtifactHashDrift()` from accepted-write helpers and update stored hashes only through `recordCheckpointResult()` / `recordTaskGroupExecution()`. |
| T-04-05 | Tampering | `context.md` capsule | mitigate | Regenerate `context.md` from normalized state plus last verification, never from chat history. |
| T-04-06 | Tampering | `drift.md` ledger | mitigate | Preserve stable headings and append timestamped bullets for assumptions, scope changes, path warnings, requirements, and approvals. |
| T-04-07 | Tampering | `buildApplyInstructions()` | mitigate | Persist `active.taskGroup` / `active.nextTaskGroup` and expose only one top-level group by default. |
</threat_model>
