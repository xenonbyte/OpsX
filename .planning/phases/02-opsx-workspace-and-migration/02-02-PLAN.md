<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| legacy repo/home filesystem -> migration planner | Potentially hostile or inconsistent paths are converted into move/create operations |
| migration plan -> execute-mode file writes | Any preflight mistake can overwrite canonical `.opsx` state or leave partial migration behind |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering / Elevation of Privilege | `lib/migrate.js`, `lib/workspace.js` | mitigate | Resolve every path against repo/home roots and reject any `path.relative()` escape before creating the plan or scaffolds. |
| T-02-02 | Tampering | `createMigrationPlan()` preflight | mitigate | Populate `abortReason` when `.opsx/` or canonical shared-home destinations already exist, and refuse any overwrite-by-default behavior. |
| T-02-03 | Denial of Service / Tampering | `runMigration()` execution ordering | mitigate | Build the full plan first, execute all moves before scaffolds, and return early on conflicts so repo/home state never lands half-migrated by design. |
| T-02-04 | Repudiation | `formatMigrationPlan()` and `runMigration()` | mitigate | Make dry-run and execute consume the same `moves` and `creates` arrays so printed output and actual mutation are mechanically aligned. |
</threat_model>
